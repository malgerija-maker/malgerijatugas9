1. Builder Pattern

Builder Pattern Builder Pattern (Pola Pembangun) adalah salah satu design pattern (pola desain) dalam pemrograman berorientasi bjek yang digunakan untuk membuat (menginstansiasi) sebuah objek kompleks secara bertahap. Pola ini masuk ke dalam kategori Creational Patterns (Pola Pembuatan). Sederhananya, Builder Pattern memisahkan proses pembuatan objek yang rumit dari representasi objek itu sendiri, sehingga proses pembuatan yang sama bisa menghasilkan representasi yang berbeda-beda.


// Nama  : M. Algerija Hertaz Paslam
// NIM   : 251410021
// Kelas : SI2A

class Config {}

class ConfigBuilder {

  ConfigBuilder host(String h) {
    print("Host : $h");
    return this;
  }

  ConfigBuilder port(int p) {
    print("Port : $p");
    return this;
  }

  Config build() {
    print("Config berhasil dibuat");
    return Config();
  }
}

void main() {

  ConfigBuilder()
      .host('localhost')
      .port(8080)
      .build();
}

2. Masalah Tanpa Extension Methods

Dalam pemrograman berorientasi objek (seperti di C# atau Kotlin), Extension Methods adalah fitur yang memungkinkan kita untuk "menambahkan" metode baru ke dalam kelas yang sudah ada tanpa harus mengubah kode sumber asli kelas tersebut, tanpa memodifikasi kelasnya langsung, dan tanpa menggunakan warisan (inheritance). Lalu, apa jadinya jika kita menghadapi situasi Tanpa Extension Methods? Mari kita bedah masalah utama yang sering muncul dan bagaimana dampaknya terhadap kualitas kode Anda.


// Nama  : M. Algerija Hertaz Paslam
// NIM   : 251410021

class StringUtils {

  static String capitalize(String text) {
    if (text.isEmpty) return text;
    return text[0].toUpperCase() + text.substring(1);
  }

  static bool isEmail(String text) {
    return text.contains('@');
  }
}

void main() {

  var name = 'john';

  var capitalized = StringUtils.capitalize(name);
  var isEmail = StringUtils.isEmail('test@mail.com');

  print('Original name : $name');
  print('Capitalized    : $capitalized');
  print('Is Email       : $isEmail');
}

3. Solusi dengan Extension Methods

Extension Methods hadir sebagai solusi elegan untuk mengatasi keterbatasan helper class konvensional. Fitur ini memungkinkan kita "menyuntikkan" metode baru ke dalam kelas yang sudah ada (bahkan kelas bawaan sistem atau library pihak ketiga yang kodenya tidak bisa kita ubah) tanpa harus melakukan pewarisan (inheritance) atau memodifikasi kode asli kelas tersebut


// Nama  : M. Algerija Hertaz Paslam
// NIM   : 251410021

extension StringExtensions on String {

  String capitalize() {
    if (isEmpty) return this;
    return this[0].toUpperCase() + substring(1);
  }

  bool get isEmail => contains('@');

  String reverse() {
    return split('').reversed.join();
  }
}

void main() {

  var name = 'john';

  var capitalized = name.capitalize();
  var isEmail = 'test@mail.com'.isEmail;
  var reversed = 'hello'.reverse();

  print('Original name : $name');
  print('Capitalized    : $capitalized');
  print('Is Email       : $isEmail');
  print('Reverse hello  : $reversed');
}

4. Sintaks Extension Methods

Extension diawali dengan kata kunci extension diikuti nama (opsional), lalu on dan tipe data yang akan diperluas. Kita bisa mendefinisikan method, getter, setter, bahkan operator overloading.


extension DateTimeExtensions on DateTime {

  // Method
  String toFormattedString() {
    return '$day/$month/$year';
  }

  // Getter
  bool get isToday {
    var now = DateTime.now();

    return year == now.year &&
           month == now.month &&
           day == now.day;
  }

  // Operator
  DateTime operator +(Duration duration) {
    return add(duration);
  }
}

void main() {

  DateTime today = DateTime.now();

  print('Tanggal hari ini        : ${today.toFormattedString()}');
  print('Apakah hari ini?        : ${today.isToday}');

  DateTime nextDay = today + Duration(days: 1);

  print('Tanggal setelah ditambah: ${nextDay.toFormattedString()}');
}

5. Extension untuk Null Safety

Di era Null Safety, Null Safety adalah fitur pada bahasa pemrograman yang dirancang untuk mencegah terjadinya salah satu kesalahan paling umum dan mematikan dalam dunia software: Null Pointer Exception (NPE) — atau sering disebut sebagai "The Billion Dollar Mistake". Secara sederhana, Null Safety memaksa compiler untuk mendeteksi potensi error akibat nilai null (data kosong/tidak ada) saat kode sedang ditulis (compile-time), bukan saat aplikasi sudah berjalan di tangan pengguna (run-time).


extension IntListExtensions on List {

  int get sum => fold(0, (total, item) => total + item);

  double get average => isEmpty ? 0 : sum / length;
}

void main() {

  var numbers = [1, 2, 3, 4, 5];

  print('Data angka      : $numbers');
  print('Jumlah data     : ${numbers.sum}');
  print('Rata-rata data  : ${numbers.average}');
}

6. Utilities untuk Dart Types - PRAKTEK 1

Ketika berbicara tentang Utilities Number (atau Number Utilities) dalam dunia pemrograman, kita merujuk pada sekumpulan fungsi pembantu (helper functions) atau kelas khusus yang digunakan untuk mempermudah manipulasi, validasi, dan pemformatan tipe data angka (int, float, double).


extension NullableStringExtensions on String? {

  String orEmpty() => this ?? '';

  bool get isNullOrEmpty => this == null || this!.isEmpty;

  bool get isNotNullOrEmpty => !isNullOrEmpty;

  String withDefault(String defaultValue) {
    return this ?? defaultValue;
  }
}

void main() {

  String? name;

  print('Hasil orEmpty()            : ${name.orEmpty()}');
  print('Hasil isNullOrEmpty        : ${name.isNullOrEmpty}');
  print('Hasil isNotNullOrEmpty     : ${name.isNotNullOrEmpty}');
  print('Hasil withDefault("Guest") : ${name.withDefault('Guest')}');
}

7. Generic Extensions

Jika Extension Method biasa digunakan untuk menambahkan fungsi ke satu tipe data spesifik (misalnya hanya untuk kelas String atau int), maka Generic Extension memungkinkan kita menambahkan fungsi baru ke banyak tipe data sekaligus secara dinamis, tanpa kehilangan kepastian tipe (type safety).


extension NumberExtensions on num {

  // Format ke Rupiah
  String toRupiah() {
    return 'Rp${toStringAsFixed(0)}';
  }

  // Cek nilai dalam range
  bool isBetween(num min, num max) {
    return this >= min && this <= max;
  }

  // Pangkat
  num power(int exponent) {

    var result = this;

    for (var i = 1; i < exponent; i++) {
      result *= this;
    }

    return result;
  }
}

void main() {

  var price = 150000;

  print('Harga dalam rupiah : ${price.toRupiah()}');
  print('Apakah antara 1-10 : ${5.isBetween(1, 10)}');
  print('Hasil 2 pangkat 3  : ${2.power(3)}');
}

8. Unnamed Extensions

Unnamed Extensions (Ekstensi Tanpa Nama) adalah sebuah fitur dalam bahasa pemrograman (paling populer dan sering dibahas di bahasa Dart) yang memungkinkan developer untuk membuat Extension Methods tanpa harus memberikan nama khusus pada blok ekstensi tersebut.


extension ListExtensions on List {

  // Mengambil index dengan aman
  T? getOrNull(int index) {
    if (index < 0 || index >= length) {
      return null;
    }

    return this[index];
  }

  // Membagi list menjadi beberapa bagian
  List> chunk(int size) {

    var chunks = >[];

    for (var i = 0; i < length; i += size) {

      chunks.add(
        sublist(i, i + size > length ? length : i + size)
      );
    }

    return chunks;
  }

  // Menghapus data duplikat
  List distinct() {

    var result = [];

    for (var item in this) {

      if (!result.contains(item)) {
        result.add(item);
      }
    }

    return result;
  }
}

void main() {

  var numbers = [1, 2, 3, 4, 5, 6, 7];

  print('Data chunk : ${numbers.chunk(3)}');

  print(
    'Data distinct : ${[1, 2, 2, 3, 3].distinct()}'
  );
}

9. Konflik Nama Extension

konflik Nama (Name Collision atau Extension Conflict) terjadi ketika ada dua atau lebih Extension Methods yang memiliki nama fungsi yang sama, ditujukan untuk tipe data yang sama, dan aktif dalam ruang lingkup (scope) yang sama. Ketika hal ini terjadi, compiler akan bingung memilih fungsi mana yang harus dijalankan, sehingga menyebabkan error kode (ambiguitas).


extension on DateTime {

  String get toYYYYMMDD {

    return '$year-'
           '${month.toString().padLeft(2, '0')}-'
           '${day.toString().padLeft(2, '0')}';
  }

  DateTime get startOfDay {
    return DateTime(year, month, day);
  }
}

void main() {

  var today = DateTime.now();

  print('Format tanggal : ${today.toYYYYMMDD}');
  print('Awal hari      : ${today.startOfDay}');
}

10. Extension untuk Custom Classes

Custom Class Extension adalah praktik menerapkan Extension Methods pada kelas-kelas yang Anda buat sendiri (bukan kelas bawaan bahasa pemrograman seperti String atau int). Meskipun Anda memiliki akses penuh untuk mengubah kode di dalam Custom Class tersebut, ada kalanya memisahkan fungsi tertentu ke dalam Extension jauh lebih baik demi menjaga arsitektur kode tetap bersih dan rapi.


// Extension pertama
extension StringCaseExtensions on String {

  String capitalize() {
    return
      "${this[0].toUpperCase()}${substring(1)}";
  }
}

// Extension kedua
extension TextUtilsExtensions on String {

  String capitalize() {
    return
      ">>> ${toUpperCase()} <<<";
  }
}

void main() {

  var text = 'hello';

  // Menggunakan extension pertama
  print(
    StringCaseExtensions(text)
        .capitalize()
  );

  // Menggunakan extension kedua
  print(
    TextUtilsExtensions(text)
        .capitalize()
  );
}

11. Validator Extensions - PRAKTEK 2

Validator Extensions adalah sebuah pola desain praktis di mana kita menggunakan Extension Methods untuk menambahkan fungsi validasi (seperti pengecekan format data, kelayakan input, atau aturan bisnis) langsung ke tipe data bawaan atau custom class. Daripada membuat kelas utilitas terpisah seperti FormValidator.validateEmail(input), pola ini memungkinkan kita menulis kode yang sangat intuitif seperti input.isValidEmail().


class Product {

  final String name;
  final double price;

  Product(this.name, this.price);
}

// Extension format Rupiah
extension DoubleRupiahFormatter on double {

  String toRupiah() {

    final intValue = toInt();

    String numberString =
        intValue.toString();

    String formattedString = '';

    int counter = 0;

    for (int i =
            numberString.length - 1;
        i >= 0;
        i--) {

      formattedString =
          numberString[i] +
          formattedString;

      counter++;

      if (counter % 3 == 0 &&
          i != 0) {

        formattedString =
            '.$formattedString';
      }
    }

    return 'Rp$formattedString';
  }
}

// Extension Product
extension ProductExtensions on Product {

  // Method diskon
  Product withDiscount(
      double percentage) {

    var discountedPrice =
        price *
        (1 - percentage / 100);

    return Product(
      name,
      discountedPrice,
    );
  }

  // Getter informasi produk
  String get displayInfo =>
      '$name - ${price.toRupiah()}';

  // Getter cek harga mahal
  bool get isExpensive =>
      price > 100000;
}

void main() {

  var laptop =
      Product('Laptop', 15000000);

  var discounted =
      laptop.withDiscount(10);

  print("=== DATA LAPTOP ===");

  print(
    discounted.displayInfo
  );

  print(
    "Laptop mahal? "
    "${laptop.isExpensive}"
  );

  var keyboard =
      Product('Keyboard', 75000);

  print("\n=== DATA KEYBOARD ===");

  print(
    keyboard.displayInfo
  );

  print(
    "Keyboard mahal? "
    "${keyboard.isExpensive}"
  );
}

12. Prioritas Method Asli vs Extension

Ketika kita membuat sebuah Extension Method yang memiliki nama dan parameter yang sama persis dengan metode bawaan yang sudah ada di dalam kelas aslinya, bahasa pemrograman memiliki aturan baku untuk menentukan fungsi mana yang akan dijalankan. Aturan ini disebut dengan Prioritas Method (Method Resolution Priority).


// Extension validator String
extension StringValidators on String {

  // Validasi email
  bool get isEmail {

    return RegExp(
      r'^[\w\.-]+@([\w-]+\.)+[\w-]{2,4}$',
    ).hasMatch(this);
  }

  // Validasi nomor HP
  bool get isPhoneNumber {

    return RegExp(
      r'^[0-9]{10,13}$',
    ).hasMatch(this);
  }

  // Validasi password kuat
  bool get isStrongPassword {

    return length >= 8 &&
        contains(
          RegExp(r'[A-Z]')
        ) &&
        contains(
          RegExp(r'[a-z]')
        ) &&
        contains(
          RegExp(r'[0-9]')
        );
  }

  // Validasi URL
  bool get isUrl {

    try {

      Uri.parse(this);

      return true;

    } catch (_) {

      return false;
    }
  }
}

// Extension validator int
extension IntValidators on int {

  bool get isAdultAge =>
      this >= 18;

  bool get isPositive =>
      this > 0;

  bool get isEven =>
      this % 2 == 0;

  bool get isOdd =>
      this % 2 != 0;
}

void main() {

  var email =
      'test@example.com';

  var age = 25;

  var password =
      'Pass1234';

  var phone =
      '081234567890';

  print("=== VALIDATOR DATA ===");

  print(
    "Email valid : "
    "${email.isEmail}"
  );

  print(
    "Nomor HP valid : "
    "${phone.isPhoneNumber}"
  );

  print(
    "Password kuat : "
    "${password.isStrongPassword}"
  );

  print(
    "Usia dewasa : "
    "${age.isAdultAge}"
  );

  print(
    "Bilangan genap : "
    "${10.isEven}"
  );
}

13. Konflik Nama Antar Extension

Konflik Nama Antar Extension (Extension vs Extension Conflict) terjadi ketika dua atau lebih Extension Methods berbeda mendefinisikan nama fungsi yang sama persis untuk tipe data yang sama, dan keduanya aktif di ruang lingkup (scope) berkas yang sama.


// Extension pertama
extension StringNumber on String {

  int toInt() {
    return int.parse(this);
  }
}

// Extension kedua
extension StringDouble on String {

  double toDouble() {
    return double.parse(this);
  }
}

void main() {

  String angka = '123';

  // Menggunakan extension int
  print(
    StringNumber(angka).toInt()
  );

  // Menggunakan extension double
  print(
    StringDouble(angka).toDouble()
  );
}

14. Prioritas Method Asli vs Extension (dan Nullable)

Ketika kita menggabungkan aturan Prioritas Method dengan konsep Nullable (Null Safety), ada beberapa skenario menarik yang terjadi di balik layar. Bagaimana jika sebuah kelas memiliki metode asli bernama sama dengan extension, namun variabel yang memanggilnya berstatus nullable (bisa null)?


class Greeting {

  String say() =>
      'Hello from class';
}

// Extension
extension GreetingExt on Greeting {

  String sayExt() =>
      'Hello from extension';
}

void main() {

  Greeting? obj = Greeting();

  // Method asli class
  print(obj?.say());

  // Method extension
  print(obj?.sayExt());
}

✨ KESIMPULAN

📌 Ringkasan Methods vs Extension dalam OOP Dart

  • Methods (Instance Method): Perilaku bawaan yang didefinisikan langsung di dalam kelas. Memiliki akses penuh ke properti private.
  • Extension Methods: Mekanisme untuk menambah fungsionalitas ke tipe data yang sudah ada (bawaan atau custom) tanpa mengubah kode sumber asli.
  • Keuntungan utama: kode lebih ekspresif, readable, dan terorganisir. Contoh: nama.capitalize() lebih natural daripada StringUtils.capitalize(nama).
  • Pahami prioritas (Method Asli > Extension Non-Nullable > Extension Nullable) untuk menghindari perilaku tak terduga.
  • Extension methods mendukung prinsip Open/Closed (terbuka untuk ekstensi, tertutup untuk modifikasi) dalam OOP.
